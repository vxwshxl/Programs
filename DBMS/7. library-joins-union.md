## 1. Database & Table Setup

Copy and run this block in your MySQL command line or GUI to set up the database, create the tables, and insert sample data.

## 2. Queries and Solutions

### i) Write an SQL query to fetch the list of books borrowed by members, along with their corresponding category names. Use the INNER JOIN operation on the Books, Categories and Borrowed_Books tables.

#### SQL Query:
```sql
SELECT 
    b.title, 
    b.author, 
    c.category_name, 
    bb.member_name
FROM Borrowed_Books bb
INNER JOIN Books b ON bb.book_id = b.book_id
INNER JOIN Categories c ON b.category_id = c.category_id;
```

#### Output:
| title | author | category_name | member_name |
|---|---|---|---|
| A Brief History of Time | Stephen Hawking | Science | John |
| To Kill a Mockingbird | Harper Lee | Fiction | Jane |
| Sapiens | Yuval Noah Harari | History | John |
| Steve Jobs | Walter Isaacson | Biography | Robert |

#### Explanation:
* An `INNER JOIN` matches and returns rows only when there is a match in both joined tables. 
* By joining `Borrowed_Books` to `Books` (using `book_id`) and then to `Categories` (using `category_id`), we get a list showing only the books that have been borrowed, along with their details and categories.

### ii) Write an SQL query to find all the books that are not currently borrowed by any member. Use the LEFT JOIN operation on the Books and Borrowed_Books tables.

#### SQL Query:
```sql
SELECT 
    b.book_id, 
    b.title, 
    b.author
FROM Books b
LEFT JOIN Borrowed_Books bb ON b.book_id = bb.book_id
WHERE bb.book_id IS NULL;
```

#### Output:
| book_id | title | author |
|---|---|---|
| 102 | The Grand Design | Stephen Hawking |
| 106 | The Origin of Species | Charles Darwin |

#### Explanation:
* A `LEFT JOIN` returns all rows from the left table (`Books`), and matching rows from the right table (`Borrowed_Books`).
* If there is no match on the right side (meaning the book has never been borrowed), the columns from `Borrowed_Books` will display as `NULL`.
* `WHERE bb.book_id IS NULL` filters the output to only show these unborrowed books.

### iii) Write an SQL query to combine the following two datasets using the UNION operator:
* The names of all authors of books in the Science category.
* The names of all authors of books borrowed by the members with the name John.

#### SQL Query:
```sql
SELECT b.author
FROM Books b
JOIN Categories c ON b.category_id = c.category_id
WHERE c.category_name = 'Science'

UNION

SELECT b.author
FROM Books b
JOIN Borrowed_Books bb ON b.book_id = bb.book_id
WHERE bb.member_name = 'John';
```

#### Output:
| author |
|---|
| Stephen Hawking |
| Charles Darwin |
| Yuval Noah Harari |

#### Explanation:
* The first query retrieves authors whose books belong to the `'Science'` category (`Stephen Hawking`, `Charles Darwin`).
* The second query retrieves authors of books borrowed by `'John'` (`Stephen Hawking`, `Yuval Noah Harari`).
* The `UNION` operator combines these two result sets and removes any duplicate entries (which is why `Stephen Hawking` is only listed once).

### iv) Assume you are adding a new book to the Books table. Write one SQL statement to insert the record, ensuring that the category_id of the book matches an existing category_id in the Categories table. Explain how foreign key constraints ensure data integrity in this scenario.

#### SQL Statement:
```sql
-- Category ID 1 represents 'Science', which exists in the Categories table.
INSERT INTO Books (book_id, title, author, category_id) 
VALUES (107, 'Cosmos', 'Carl Sagan', 1);
```

#### Verification:
```sql
SELECT * FROM Books WHERE book_id = 107;
```
| book_id | title | author | category_id |
|---|---|---|---|
| 107 | Cosmos | Carl Sagan | 1 |

#### Explanation of Foreign Keys and Data Integrity:
* A **Foreign Key (FK)** constraint creates a logical link between a column in one table (e.g., `category_id` in `Books`) and the Primary Key in another table (e.g., `category_id` in `Categories`).
* If you try to insert a book with a `category_id` that does not exist (like `category_id = 99`):
  ```sql
  INSERT INTO Books (book_id, title, author, category_id) 
  VALUES (108, 'Test Book', 'Test Author', 99);
  ```
  MySQL will block this and display an error:
  `ERROR 1452 (23000): Cannot add or update a child row: a foreign key constraint fails...`
* This protects **Referential Integrity** by preventing the entry of invalid data (orphaned rows) into the database.
