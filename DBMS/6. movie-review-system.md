# Experiment 6: Movie Review System

## Aim
To create a database schema for a Movie Review System with `Movies` and `Reviews` tables, and retrieve average ratings and the most recently reviewed movies using aggregation and ordering in MySQL.

---

## 1. Database & Table Setup

Copy and run this block in your MySQL command line or GUI to set up the database, create the tables, and insert sample data.

```sql
-- Step 1: Create and use a new database
CREATE DATABASE IF NOT EXISTS dbms_lab_q6;
USE dbms_lab_q6;

-- Step 2: Drop existing tables (in reverse order of foreign keys)
DROP TABLE IF EXISTS Reviews;
DROP TABLE IF EXISTS Movies;

-- Step 3: Create the Movies table
CREATE TABLE Movies (
    movie_id INT PRIMARY KEY AUTO_INCREMENT,
    title VARCHAR(150) NOT NULL,
    genre VARCHAR(50) NOT NULL,
    release_year INT NOT NULL
);

-- Step 4: Create the Reviews table (with a rating check constraint)
CREATE TABLE Reviews (
    review_id INT PRIMARY KEY AUTO_INCREMENT,
    movie_id INT,
    reviewer_name VARCHAR(100) NOT NULL,
    rating INT NOT NULL CHECK (rating BETWEEN 1 AND 5),
    review_date DATE NOT NULL,
    FOREIGN KEY (movie_id) REFERENCES Movies(movie_id) ON DELETE CASCADE
);
```

## 2. Queries and Solutions

### Query 1: Write a query to list all movie titles along with their genres and average rating.

#### SQL Query:
```sql
SELECT m.title, m.genre, AVG(r.rating) AS average_rating
FROM Movies m
LEFT JOIN Reviews r ON m.movie_id = r.movie_id
GROUP BY m.movie_id, m.title, m.genre;
```

#### Output:
| title | genre | average_rating |
|---|---|---|
| Inception | Sci-Fi | 4.5000 |
| The Dark Knight | Action | 5.0000 |
| Interstellar | Sci-Fi | 3.5000 |
| Pulp Fiction | Crime | 4.5000 |
| The Shawshank Redemption | Drama | 5.0000 |

#### Explanation:
* We use a `LEFT JOIN` to combine the `Movies` table with the `Reviews` table. A `LEFT JOIN` ensures that if a movie has no reviews, it still appears in the output (with a `NULL` rating).
* `AVG(r.rating)` calculates the mathematical average of all ratings given to each movie.
* `GROUP BY` aggregates the reviews based on each unique movie.

---

### Query 2: Write a query to list the 3 most recently reviewed movies. Use the LIMIT clause.

#### SQL Query:
```sql
SELECT m.title, MAX(r.review_date) AS most_recent_review_date
FROM Movies m
JOIN Reviews r ON m.movie_id = r.movie_id
GROUP BY m.movie_id, m.title
ORDER BY most_recent_review_date DESC
LIMIT 3;
```

#### Output:
| title | most_recent_review_date |
|---|---|
| The Shawshank Redemption | 2026-05-27 |
| Interstellar | 2026-05-22 |
| The Dark Knight | 2026-05-20 |

#### Explanation:
* `MAX(r.review_date)` finds the date of the most recent review submitted for each movie.
* `GROUP BY` ensures we only have one row per movie title.
* `ORDER BY most_recent_review_date DESC` sorts the list from the most recent review date down to the oldest.
* `LIMIT 3` selects only the top 3 rows from the sorted result.
