# Experiment 9: Student Triggers (Validation & Auditing)

## Aim
To create database triggers in MySQL to automatically handle data auditing (logging), enforce validation checks (preventing invalid age and marks), and automatically generate timestamps on inserts.

---

## 1. Table Setup

Copy and run this block in your MySQL client to create the tables.

```sql
CREATE TABLE students (
    student_id INT AUTO_INCREMENT PRIMARY KEY,
    student_name VARCHAR(100) NOT NULL,
    age INT NOT NULL,
    marks DECIMAL(5, 2) NOT NULL,
    created_at TIMESTAMP NULL DEFAULT NULL
);

CREATE TABLE student_log (
    log_id INT AUTO_INCREMENT PRIMARY KEY,
    student_id INT,
    student_name VARCHAR(100),
    logged_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

---

## 2. Trigger Creations

In MySQL, triggers are created to run automatically **BEFORE** or **AFTER** a database event like `INSERT`, `UPDATE`, or `DELETE`. 

---

### i) Create a trigger to log inserts. Create a table student_log. Write a trigger that logs every insert operation on students.

#### SQL Statement to Create Trigger:
```sql
DELIMITER //

CREATE TRIGGER after_student_insert
AFTER INSERT ON students
FOR EACH ROW
BEGIN
    INSERT INTO student_log (student_id, student_name)
    VALUES (NEW.student_id, NEW.student_name);
END //

DELIMITER ;
```

---

### ii) & iii) & iv) Create a trigger that:
* **(ii)** Prevents inserting a student with `age < 18`.
* **(iii)** Automatically sets the `created_at` column when a new record is inserted.
* **(iv)** Stops insertion if `marks < 0`.

#### SQL Statement to Create Trigger:
```sql
DELIMITER //

CREATE TRIGGER before_student_insert
BEFORE INSERT ON students
FOR EACH ROW
BEGIN
    SET NEW.created_at = NOW();

    IF NEW.age < 18 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'ERROR: Student age must be 18 or older.';
    END IF;

    IF NEW.marks < 0.00 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'ERROR: Marks cannot be less than 0.';
    END IF;
END //

DELIMITER ;
```

---

### iv) Write a trigger that stops UPDATE if marks < 0.

#### SQL Statement to Create Trigger:
```sql
DELIMITER //

CREATE TRIGGER before_student_update
BEFORE UPDATE ON students
FOR EACH ROW
BEGIN
    IF NEW.marks < 0.00 THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'ERROR: Update blocked. Marks cannot be less than 0.';
    END IF;
END //

DELIMITER ;
```

## 3. Testing and Verification

### Scenario 1: Insert a valid student

#### SQL Query:
```sql
INSERT INTO students (student_name, age, marks) VALUES ('Aarav Baruah', 20, 85.50);
```

#### Verification Queries:
```sql
SELECT * FROM students;
```
| student_id | student_name | age | marks | created_at |
|---|---|---|---|---|
| 1 | Aarav Baruah | 20 | 85.50 | 2026-06-01 19:30:00 |

*(Note: The `created_at` timestamp was automatically set to the current date/time by our trigger)*

```sql
SELECT * FROM student_log;
```
| log_id | student_id | student_name | logged_at |
|---|---|---|---|
| 1 | 1 | Aarav Baruah | 2026-06-01 19:30:00 |

*(Note: The log row was automatically added to `student_log` by the insert trigger)*
