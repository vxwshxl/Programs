## 2. Procedures and Solutions

Stored procedures use statements inside `BEGIN ... END`. In MySQL, we must temporarily change the statement delimiter (usually from `;` to `//` or `$$`) while writing the procedure, so that MySQL does not try to execute the query prematurely when it sees the semicolons inside the body.

---

### i) Write a procedure to add two numbers and return the sum.

#### SQL Statement to Create Procedure:
```sql
DELIMITER //

CREATE PROCEDURE AddTwoNumbers(
    IN num1 INT,
    IN num2 INT,
    OUT sum_val INT
)
BEGIN
    SET sum_val = num1 + num2;
END //

DELIMITER ;
```

#### SQL Command to Run (Execute):
```sql
-- 1. Initialize a variable to hold the output
SET @sum_result = 0;

-- 2. Call the procedure
CALL AddTwoNumbers(25, 45, @sum_result);

-- 3. Display the result
SELECT @sum_result AS 'Sum Result';
```

#### Output:
| Sum Result |
|---|
| 70 |

#### Explanation:
* `IN num1` and `IN num2` are the input values we pass to the procedure (`25` and `45`).
* `OUT sum_val` is the variable that will return the calculated result back to us.
* We use `CALL ProcedureName(arguments...)` to execute a procedure.

### ii) Write a procedure to return the maximum salary from a given employee table. Consider the employee table.

#### SQL Statement to Create Procedure:
```sql
DELIMITER //

CREATE PROCEDURE GetMaxSalary(
    OUT max_sal DECIMAL(10, 2)
)
BEGIN
    SELECT MAX(salary) INTO max_sal FROM Employee;
END //

DELIMITER ;
```

#### SQL Command to Run (Execute):
```sql
-- 1. Initialize a variable to hold the output
SET @max_salary_val = 0.00;

-- 2. Call the procedure
CALL GetMaxSalary(@max_salary_val);

-- 3. Display the result
SELECT @max_salary_val AS 'Maximum Salary';
```

#### Output:
| Maximum Salary |
|---|
| 95000.00 |

#### Explanation:
* `SELECT MAX(salary) INTO max_sal FROM Employee` finds the maximum salary from the `Employee` table and directly saves it into our output parameter `max_sal`.

### iii) Create a procedure to count the total number of a particular subject. Provide the subject name as an input parameter and get the total count through an OUT parameter.

#### SQL Statement to Create Procedure:
```sql
DELIMITER //

CREATE PROCEDURE CountSubjectCourses(
    IN sub_name VARCHAR(100),
    OUT total_count INT
)
BEGIN
    SELECT COUNT(*) INTO total_count 
    FROM StudentCourses 
    WHERE subject_name = sub_name;
END //

DELIMITER ;
```

#### SQL Command to Run (Execute):
```sql
-- 1. Initialize a variable to hold the output count
SET @dbms_count = 0;

-- 2. Call the procedure for the subject 'DBMS'
CALL CountSubjectCourses('DBMS', @dbms_count);

-- 3. Display the result
SELECT 'DBMS' AS 'Subject', @dbms_count AS 'Total Count';
```

#### Output:
| Subject | Total Count |
|---|---|
| DBMS | 3 |

#### Explanation:
* We pass `'DBMS'` as the input parameter `sub_name`.
* The procedure filters the rows in `StudentCourses` where `subject_name = 'DBMS'`, counts them using `COUNT(*)`, and stores the count in our output variable `total_count`.
* We retrieve the result by selecting the session variable `@dbms_count`.
