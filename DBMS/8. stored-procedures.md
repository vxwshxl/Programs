## 2. Procedures and Solutions

Stored procedures are created using the `CREATE PROCEDURE` statement. We temporarily modify the execution delimiter (from `;` to `//`) to define the procedure block as a single unit without the internal query semicolons executing prematurely.

---

### i) Write a procedure to add two numbers and return the sum.

#### SQL Statement to Create Procedure:
```sql
DELIMITER //

CREATE PROCEDURE AddTwoNumbers(
    IN num1 INT,
    IN num2 INT,
    OUT sum_val INT
)
BEGIN
    SET sum_val = num1 + num2;
END //

DELIMITER ;
```

#### SQL Command to Run (Execute):
```sql
SET @sum_result = 0;
CALL AddTwoNumbers(25, 45, @sum_result);
SELECT @sum_result AS 'Sum Result';
```

#### Output:
| Sum Result |
|---|
| 70 |

#### Explanation:
* `IN num1` and `IN num2` are parameters that accept incoming integer arguments.
* `OUT sum_val` returns the computed addition back to the caller.
* `@sum_result` is a session variable used to hold the returned value.

### ii) Write a procedure to return the maximum salary from a given employee table. Consider the employee table.

#### SQL Statement to Create Procedure:
```sql
DELIMITER //

CREATE PROCEDURE GetMaxSalary(
    OUT max_sal DECIMAL(10, 2)
)
BEGIN
    SELECT MAX(salary) INTO max_sal FROM Employee;
END //

DELIMITER ;
```

#### SQL Command to Run (Execute):
```sql
SET @max_salary_val = 0.00;
CALL GetMaxSalary(@max_salary_val);
SELECT @max_salary_val AS 'Maximum Salary';
```

#### Output:
| Maximum Salary |
|---|
| 95000.00 |

#### Explanation:
* The procedure runs `SELECT MAX(salary) FROM Employee` and uses the `INTO` keyword to assign the result directly to our output parameter `max_sal`.

### iii) Create a procedure to count the total number of a particular subject. Provide the subject name as an input parameter and get the total count through an OUT parameter.

#### SQL Statement to Create Procedure:
```sql
DELIMITER //

CREATE PROCEDURE CountSubjectCourses(
    IN sub_name VARCHAR(100),
    OUT total_count INT
)
BEGIN
    SELECT COUNT(*) INTO total_count 
    FROM StudentCourses 
    WHERE subject_name = sub_name;
END //

DELIMITER ;
```

#### SQL Command to Run (Execute):
```sql
SET @dbms_count = 0;
CALL CountSubjectCourses('DBMS', @dbms_count);
SELECT 'DBMS' AS 'Subject', @dbms_count AS 'Total Count';
```

#### Output:
| Subject | Total Count |
|---|---|
| DBMS | 3 |

#### Explanation:
* We pass `'DBMS'` as the input parameter `sub_name`.
* The procedure filters rows in the `StudentCourses` table where `subject_name = 'DBMS'`, counts them, and returns that count through the `total_count` variable.
