# Experiment 5: Customer Purchase Database

## Aim
To design a database schema for customer purchases using `Customer`, `Item`, and `Purchase` tables with foreign key constraints, and query data for purchase aggregates, monthly filtering, and join checks.

---

## 1. Database & Table Setup

Copy and run this block in your MySQL command line or GUI to set up the database, create the tables, and insert sample data.

```sql
-- Step 1: Create and use a new database
CREATE DATABASE IF NOT EXISTS dbms_lab_q5;
USE dbms_lab_q5;

-- Step 2: Drop existing tables (in reverse order of foreign keys)
DROP TABLE IF EXISTS Purchase;
DROP TABLE IF EXISTS Item;
DROP TABLE IF EXISTS Customer;

-- Step 3: Create the Customer table
CREATE TABLE Customer (
    c_id INT PRIMARY KEY,
    c_name VARCHAR(100) NOT NULL,
    address VARCHAR(200) NOT NULL
);

-- Step 4: Create the Item table
CREATE TABLE Item (
    i_code INT PRIMARY KEY,
    i_name VARCHAR(100) NOT NULL,
    price DECIMAL(10, 2) NOT NULL
);

-- Step 5: Create the Purchase table with foreign keys
CREATE TABLE Purchase (
    p_id INT PRIMARY KEY,
    c_id INT,
    i_code INT,
    qty INT NOT NULL,
    date DATE NOT NULL,
    FOREIGN KEY (c_id) REFERENCES Customer(c_id) ON DELETE CASCADE,
    FOREIGN KEY (i_code) REFERENCES Item(i_code) ON DELETE CASCADE
);
```

## 2. Queries and Solutions

### a) Find the name of the customer who has done the maximum purchase (in terms of total money spent).

#### SQL Query:
```sql
SELECT c.c_name, SUM(p.qty * i.price) AS total_spent
FROM Customer c
JOIN Purchase p ON c.c_id = p.c_id
JOIN Item i ON p.i_code = i.i_code
GROUP BY c.c_id, c.c_name
ORDER BY total_spent DESC
LIMIT 1;
```

#### Output:
| c_name | total_spent |
|---|---|
| Barnali Sen | 17000.00 |

#### Explanation:
* We join the three tables to associate customer names with purchase quantities and item prices.
* `SUM(p.qty * i.price)` calculates the total spending for each customer.
* `GROUP BY` aggregates the purchases per customer.
* `ORDER BY total_spent DESC LIMIT 1` sorts the customers from highest spender to lowest and takes only the top-most row.

---

### b) Display the name of the item that has been purchased maximum number of times (maximum sum of quantity) in the month of February.

#### SQL Query:
```sql
SELECT i.i_name, SUM(p.qty) AS total_qty_purchased
FROM Item i
JOIN Purchase p ON i.i_code = p.i_code
WHERE MONTH(p.date) = 2
GROUP BY i.i_code, i.i_name
ORDER BY total_qty_purchased DESC
LIMIT 1;
```

#### Output:
| i_name | total_qty_purchased |
|---|---|
| Wireless Mouse | 3 |

#### Explanation:
* `WHERE MONTH(p.date) = 2` filters purchase transactions to only those that happened in the month of February.
* `SUM(p.qty)` adds up the quantities of each item sold in February.
* `ORDER BY total_qty_purchased DESC LIMIT 1` finds the single item with the highest sum of quantities.

---

### c) Display the name of the customer who did not purchase any item.

#### SQL Query:
```sql
SELECT c_name 
FROM Customer 
WHERE c_id NOT IN (
    SELECT DISTINCT c_id 
    FROM Purchase
);
```

#### Output:
| c_name |
|---|
| Deepankar Roy |

#### Explanation:
* The subquery `SELECT DISTINCT c_id FROM Purchase` returns the IDs of all customers who have completed at least one transaction (`1, 2, 3`).
* The outer query selects names from the `Customer` table where the `c_id` is NOT in that list, showing who hasn't bought anything yet (Deepankar Roy).
