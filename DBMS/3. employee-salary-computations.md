# Experiment 3: Employee Salary Computations & Aggregations

## Aim
To create an `Employee` table with basic pay details, calculate dynamic columns (DA, TA, HRA, TAX, GROSS, and NET) using mathematical operations, and perform aggregate functions (`COUNT`, `MAX`).

## 2. Queries and Solutions

You can copy and run each query one by one.

### i) Find out gross salary and net salary.
* **DA** = 60% of Basic Pay
* **TA** = 20% of Basic Pay
* **HRA** = 10% of Basic Pay
* **TAX** = 15% of Basic Pay
* **GROSS** = Basic + DA + TA + HRA
* **NET** = GROSS - TAX

#### SQL Query:
```sql
SELECT 
    e_name, 
    designation, 
    basic_pay,
    (basic_pay * 0.60) AS DA,
    (basic_pay * 0.20) AS TA,
    (basic_pay * 0.10) AS HRA,
    (basic_pay * 0.15) AS TAX,
    (basic_pay + (basic_pay * 0.60) + (basic_pay * 0.20) + (basic_pay * 0.10)) AS GROSS,
    ((basic_pay + (basic_pay * 0.60) + (basic_pay * 0.20) + (basic_pay * 0.10)) - (basic_pay * 0.15)) AS NET
FROM Employee;
```

#### Output:
| e_name | designation | basic_pay | DA | TA | HRA | TAX | GROSS | NET |
|---|---|---|---|---|---|---|---|---|
| Ramesh Kumar | Developer | 7000.00 | 4200.00 | 1400.00 | 700.00 | 1050.00 | 13300.00 | 12250.00 |
| Sita Sharma | HR Assistant | 4000.00 | 2400.00 | 800.00 | 400.00 | 600.00 | 7600.00 | 7000.00 |
| Ganesh Das | Office Assistant | 3000.00 | 1800.00 | 600.00 | 300.00 | 450.00 | 5700.00 | 5250.00 |
| Karan Johar | Intern | 2000.00 | 1200.00 | 400.00 | 200.00 | 300.00 | 3800.00 | 3500.00 |
| Sunita Sen | Project Lead | 10000.00 | 6000.00 | 2000.00 | 1000.00 | 1500.00 | 19000.00 | 17500.00 |

#### Explanation:
* Dynamic calculations are done using simple multiplication: `basic_pay * percentage`.
* `AS` is used to give friendly names to the computed columns (column aliasing).

### ii) Find the total number of employees.

#### SQL Query:
```sql
SELECT COUNT(*) AS total_employees 
FROM Employee;
```

#### Output:
| total_employees |
|---|
| 5 |

#### Explanation:
* `COUNT(*)` is an aggregate function that returns the total count of rows in the table.

---

### iii) Find the total number of employees having gross salary between 5000 and 15000.
*(Note: Gross salary is calculated as `basic_pay * 1.9`)*

#### SQL Query:
```sql
SELECT COUNT(*) AS total_count 
FROM Employee 
WHERE (basic_pay * 1.9) BETWEEN 5000.00 AND 15000.00;
```

#### Output:
| total_count |
|---|
| 3 |

#### Explanation:
* It calculates each employee's gross salary on the fly, checks if it lies between 5000 and 15000, and counts how many matching employees there are. (In this case: Ramesh, Sita, and Ganesh are in range).

---

### iv) Find the maximum gross salary.

#### SQL Query:
```sql
SELECT MAX(basic_pay * 1.9) AS max_gross_salary 
FROM Employee;
```

#### Output:
| max_gross_salary |
|---|
| 19000.00 |

#### Explanation:
* `MAX()` finds the highest value in the calculated gross salary column.

---

### v) Display the details of the employee who gets the maximum gross salary.

#### SQL Query:
```sql
SELECT 
    emp_id, 
    e_name, 
    designation, 
    basic_pay,
    (basic_pay * 1.9) AS gross_salary 
FROM Employee 
WHERE basic_pay = (SELECT MAX(basic_pay) FROM Employee);
```

#### Output:
| emp_id | e_name | designation | basic_pay | gross_salary |
|---|---|---|---|---|
| 5 | Sunita Sen | Project Lead | 10000.00 | 19000.00 |

#### Explanation:
* A **subquery** `(SELECT MAX(basic_pay) FROM Employee)` is evaluated first. It returns `10000.00`.
* The outer query then fetches details of the employee whose `basic_pay` is equal to `10000.00`.
