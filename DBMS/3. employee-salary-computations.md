# Experiment 3: Employee Salary Computations & Aggregations

## Aim
To create an `Employee` table with basic pay details, calculate dynamic columns (DA, TA, HRA, TAX, GROSS, and NET) using mathematical operations, and perform aggregate functions (`COUNT`, `MAX`).

---

## 1. Table Setup

Copy and run this block in your MySQL client to create the table and insert the sample data.

```sql
CREATE TABLE Employee (
    emp_id INT AUTO_INCREMENT PRIMARY KEY,
    e_name VARCHAR(100) NOT NULL,
    designation VARCHAR(50) NOT NULL,
    basic_pay DECIMAL(10, 2) NOT NULL
```

## 2. Queries and Solutions

### i) Find out gross salary and net salary.
* **DA** = 60% of Basic Pay
* **TA** = 20% of Basic Pay
* **HRA** = 10% of Basic Pay
* **TAX** = 15% of Basic Pay
* **GROSS** = Basic + DA + TA + HRA
* **NET** = GROSS - TAX

#### SQL Query:
```sql
SELECT 
    e_name, 
    designation, 
    basic_pay,
    (basic_pay * 0.60) AS DA,
    (basic_pay * 0.20) AS TA,
    (basic_pay * 0.10) AS HRA,
    (basic_pay * 0.15) AS TAX,
    (basic_pay + (basic_pay * 0.60) + (basic_pay * 0.20) + (basic_pay * 0.10)) AS GROSS,
    ((basic_pay + (basic_pay * 0.60) + (basic_pay * 0.20) + (basic_pay * 0.10)) - (basic_pay * 0.15)) AS NET
FROM Employee;
```

#### Output:
| e_name | designation | basic_pay | DA | TA | HRA | TAX | GROSS | NET |
|---|---|---|---|---|---|---|---|---|
| Ramesh Kumar | Developer | 7000.00 | 4200.00 | 1400.00 | 700.00 | 1050.00 | 13300.00 | 12250.00 |
| Sita Sharma | HR Assistant | 4000.00 | 2400.00 | 800.00 | 400.00 | 600.00 | 7600.00 | 7000.00 |
| Ganesh Das | Office Assistant | 3000.00 | 1800.00 | 600.00 | 300.00 | 450.00 | 5700.00 | 5250.00 |
| Karan Johar | Intern | 2000.00 | 1200.00 | 400.00 | 200.00 | 300.00 | 3800.00 | 3500.00 |
| Sunita Sen | Project Lead | 10000.00 | 6000.00 | 2000.00 | 1000.00 | 1500.00 | 19000.00 | 17500.00 |

#### Explanation:
* Dynamic calculations are computed inline using multiplication: `basic_pay * percentage`.
* `AS` is used to assign aliases (friendly header names) to each computed column.

### ii) Find the total number of employees.

#### SQL Query:
```sql
SELECT COUNT(*) AS total_employees FROM Employee;
```

#### Output:
| total_employees |
|---|
| 5 |

#### Explanation:
* `COUNT(*)` counts and returns the total number of records (rows) in the table.

---

### iii) Find the total number of employees having gross salary between 5000 and 15000.
*(Note: Gross salary is basic_pay * 1.9)*

#### SQL Query:
```sql
SELECT COUNT(*) AS total_count FROM Employee WHERE (basic_pay * 1.9) BETWEEN 5000.00 AND 15000.00;
```

#### Output:
| total_count |
|---|
| 3 |

#### Explanation:
* We calculate the gross salary inline using `(basic_pay * 1.9)`.
* We filter rows using the `BETWEEN` operator and count the total number of matches.

---

### iv) Find the maximum gross salary.

#### SQL Query:
```sql
SELECT MAX(basic_pay * 1.9) AS max_gross_salary FROM Employee;
```

#### Output:
| max_gross_salary |
|---|
| 19000.00 |

#### Explanation:
* `MAX()` processes the calculated gross salaries across all rows and returns the single highest value.

---

### v) Display the details of the employee who gets the maximum gross salary.

#### SQL Query:
```sql
SELECT 
    emp_id, 
    e_name, 
    designation, 
    basic_pay,
    (basic_pay * 1.9) AS gross_salary 
FROM Employee 
WHERE basic_pay = (SELECT MAX(basic_pay) FROM Employee);
```

#### Output:
| emp_id | e_name | designation | basic_pay | gross_salary |
|---|---|---|---|---|
| 5 | Sunita Sen | Project Lead | 10000.00 | 19000.00 |

#### Explanation:
* The subquery `(SELECT MAX(basic_pay) FROM Employee)` executes first and returns `10000.00`.
* The outer query then retrieves and displays the full details of any employee whose `basic_pay` is equal to that maximum value.
