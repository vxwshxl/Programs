## Aim

## 2. Queries and Solutions

### i) Find the authors of the books that have not been issued.

#### SQL Query:
```sql
SELECT DISTINCT author 
FROM Books 
WHERE b_id NOT IN (
    SELECT DISTINCT b_id 
    FROM Issue_Return
);
```

#### Output:
| author |
|---|
| Alfred Aho |

#### Explanation:
* The subquery `SELECT DISTINCT b_id FROM Issue_Return` gets a list of all book IDs that have been issued at least once (`1, 2, 3, 4, 6`).
* The outer query retrieves authors from the `Books` table where `b_id` is NOT in that list (which leaves `b_id = 5`, authored by Alfred Aho).

---

### ii) Display the member IDs and number of books issued to that member.
*(Assume that if a book in the Issue_Return relation does not have a return date, then it is issued.)*

#### SQL Query:
```sql
SELECT m_id, COUNT(*) AS active_issued_books
FROM Issue_Return
WHERE r_date IS NULL
GROUP BY m_id;
```

#### Output:
| m_id | active_issued_books |
|---|---|
| 203 | 1 |
| 201 | 1 |
| 204 | 1 |

#### Explanation:
* `WHERE r_date IS NULL` selects only the rows where the book is currently out (not yet returned).
* `GROUP BY m_id` groups the active issues by each member.
* `COUNT(*)` counts how many books are issued to each of those grouped members.

### iii) Find the book that has been issued the minimum number of times.
*(Note: To be robust, this includes books issued 0 times)*

#### SQL Query:
```sql
SELECT b.b_id, b.b_name, COUNT(ir.b_id) AS issue_count
FROM Books b
LEFT JOIN Issue_Return ir ON b.b_id = ir.b_id
GROUP BY b.b_id, b.b_name
HAVING issue_count = (
    SELECT MIN(cnt)
    FROM (
        SELECT COUNT(ir2.b_id) AS cnt
        FROM Books b2
        LEFT JOIN Issue_Return ir2 ON b2.b_id = ir2.b_id
        GROUP BY b2.b_id
    ) AS sub
);
```

#### Output:
| b_id | b_name | issue_count |
|---|---|---|
| 5 | Compilers: Principles, Techniques, and Tools | 0 |

#### Explanation:
* A `LEFT JOIN` matches all books, even those with `0` issues.
* The subquery calculates the number of times each book was issued and finds the minimum count (`0`).
* The `HAVING` clause filters the grouped books to display only the ones matching that minimum count of `0`.

### iv) Display the names and author of the books that have been issued at any time to members whose name begins with 'R'.

#### SQL Query:
```sql
SELECT DISTINCT b.b_name, b.author
FROM Books b
JOIN Issue_Return ir ON b.b_id = ir.b_id
JOIN Members m ON ir.m_id = m.m_id
WHERE m.m_name LIKE 'R%';
```

#### Output:
| b_name | author |
|---|---|
| Database System Concepts | S. Sudarshan |
| Introduction to Algorithms | Thomas H. Cormen |
| Operating System Concepts | Avi Silberschatz |
| Computer Networks | Andrew S. Tanenbaum |
| Software Engineering | Ian Sommerville |

#### Explanation:
* We perform `JOIN` operations across all three tables to link books to the members who borrowed them.
* `WHERE m.m_name LIKE 'R%'` filters for members whose names start with the letter `'R'`.
* `DISTINCT` removes any duplicate book rows if a book was borrowed multiple times by 'R' members.

### v) Display the name and cost of those books that have been issued to any member whose date of birth is less than 01-01-1989, but not been issued to any member having the birth date equal to or greater than 01-01-1989.

#### SQL Query:
```sql
SELECT b_name, cost
FROM Books
WHERE b_id IN (
    -- Set 1: Book IDs issued to members born before 1989
    SELECT ir.b_id 
    FROM Issue_Return ir
    JOIN Members m ON ir.m_id = m.m_id
    WHERE m.birthdate < '1989-01-01'
)
AND b_id NOT IN (
    -- Set 2: Book IDs issued to members born on or after 1989
    SELECT ir.b_id 
    FROM Issue_Return ir
    JOIN Members m ON ir.m_id = m.m_id
    WHERE m.birthdate >= '1989-01-01'
);
```

#### Output:
| b_name | cost |
|---|---|
| Operating System Concepts | 1500.00 |
| Software Engineering | 950.00 |

#### Explanation:
* This query uses standard set subtraction using `IN` and `NOT IN`.
* First, it finds books that were borrowed by members born before 1989 (Rohan, Priya, Amit).
* Then, it excludes any books that were *also* borrowed by members born in 1989 or later (Rahul, Riya).
* Only books matching this strict condition are listed.
