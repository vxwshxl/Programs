# Experiment 4: Library Management System

## Aim
To create a relational database for a library consisting of `Books`, `Members`, and `Issue_Return` tables, and write standard SQL queries with joins, subqueries, and set differences.

---

    b_id INT PRIMARY KEY,
    b_name VARCHAR(150) NOT NULL,

## 2. Queries and Solutions

### i) Find the authors of the books that have not been issued.

#### SQL Query:
```sql
SELECT DISTINCT author 
FROM Books 
WHERE b_id NOT IN (
    SELECT DISTINCT b_id 
    FROM Issue_Return
);
```

#### Output:
| author |
|---|
| Alfred Aho |

#### Explanation:
* The subquery `SELECT DISTINCT b_id FROM Issue_Return` compiles all book IDs that have been borrowed (`1, 2, 3, 4, 6`).
* The outer query selects the authors of any books whose `b_id` is not in that list (which matches `b_id = 5`, authored by Alfred Aho).

---

### ii) Display the member IDs and number of books issued to that member.
*(Assume that if a book in the Issue_Return relation does not have a return date, then it is issued.)*

#### SQL Query:
```sql
SELECT m_id, COUNT(*) AS active_issued_books
FROM Issue_Return
WHERE r_date IS NULL
GROUP BY m_id;
```

#### Output:
| m_id | active_issued_books |
|---|---|
| 203 | 1 |
| 201 | 1 |
| 204 | 1 |

#### Explanation:
* `WHERE r_date IS NULL` selects only the transactions representing books that are currently checked out.
* `GROUP BY m_id` separates the active transactions by member ID.
* `COUNT(*)` counts the number of occurrences (active borrows) for each member ID.

### iii) Find the book that has been issued the minimum number of times.

#### SQL Query:
```sql
SELECT b.b_id, b.b_name, COUNT(ir.b_id) AS issue_count
FROM Books b
LEFT JOIN Issue_Return ir ON b.b_id = ir.b_id
GROUP BY b.b_id, b.b_name
HAVING issue_count = (
    SELECT MIN(cnt)
    FROM (
        SELECT COUNT(ir2.b_id) AS cnt
        FROM Books b2
        LEFT JOIN Issue_Return ir2 ON b2.b_id = ir2.b_id
        GROUP BY b2.b_id
    ) AS sub
);
```

#### Output:
| b_id | b_name | issue_count |
|---|---|---|
| 5 | Compilers: Principles, Techniques, and Tools | 0 |

#### Explanation:
* The `LEFT JOIN` makes sure that all books appear, including those with `0` issues.
* The nested subqueries compute the issue count of all books and retrieve the overall lowest value (`0`).
* `HAVING` filters the results to only include books with an issue count matching that minimum value.

### iv) Display the names and author of the books that have been issued at any time to members whose name begins with 'R'.

#### SQL Query:
```sql
SELECT DISTINCT b.b_name, b.author
FROM Books b
JOIN Issue_Return ir ON b.b_id = ir.b_id
JOIN Members m ON ir.m_id = m.m_id
WHERE m.m_name LIKE 'R%';
```

#### Output:
| b_name | author |
|---|---|
| Database System Concepts | S. Sudarshan |
| Introduction to Algorithms | Thomas H. Cormen |
| Operating System Concepts | Avi Silberschatz |
| Computer Networks | Andrew S. Tanenbaum |
| Software Engineering | Ian Sommerville |

#### Explanation:
* We join the three tables together to link books to the members who borrowed them.
* `WHERE m.m_name LIKE 'R%'` filters results for members whose name begins with 'R'.
* `DISTINCT` eliminates duplicate rows if a book was borrowed multiple times by members starting with 'R'.

### v) Display the name and cost of those books that have been issued to any member whose date of birth is less than 01-01-1989, but not been issued to any member having the birth date equal to or greater than 01-01-1989.

#### SQL Query:
```sql
SELECT b_name, cost
FROM Books
WHERE b_id IN (
    SELECT ir.b_id 
    FROM Issue_Return ir
    JOIN Members m ON ir.m_id = m.m_id
    WHERE m.birthdate < '1989-01-01'
)
AND b_id NOT IN (
    SELECT ir.b_id 
    FROM Issue_Return ir
    JOIN Members m ON ir.m_id = m.m_id
    WHERE m.birthdate >= '1989-01-01'
);
```

#### Output:
| b_name | cost |
|---|---|
| Operating System Concepts | 1500.00 |
| Software Engineering | 950.00 |

#### Explanation:
* The first subquery gathers book IDs issued to members born before 1989.
* The second subquery gathers book IDs issued to members born on or after 1989.
* `AND b_id NOT IN` performs set subtraction, leaving only the books issued to the older group and never borrowed by the younger group.
